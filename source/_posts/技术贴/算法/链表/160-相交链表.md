**注意：**

- 如果两个链表没有交点，返回 `null`.
- 在返回结果后，两个链表仍须保持原有的结构。
- 可假定整个链表结构中没有循环。
- 程序尽量满足 O(*n*) 时间复杂度，且仅用 O(*1*) 内存。

## 解题思路及代码

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */

// 暴力法 
// 对于每一个A链表的节点,遍历B链表是否有与之相同的节点
// 时间复杂度O(MN), 空间复杂度O(1)
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    tailA, tailB := headA, headB
    for tailA != nil {
        tailB = headB
        for tailB != nil {
            if tailA == tailB {
                return tailA
            }
            tailB = tailB.Next
        }
        tailA = tailA.Next
    }
    return nil
}


// 方法二 Hash表计数法
// 时间复杂度O(M+N), 空间复杂度O(M)/O(N)
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    tailA, tailB := headA, headB
    countMap := make(map[*ListNode]int, 0)
    // 记录A链表的出现次数
    for tailA != nil {
        countMap[tailA] = 1
        tailA = tailA.Next
    }
    // 遍历B链表
    for tailB != nil {
        if _, ok := countMap[tailB];ok {
            return tailB
        }
        tailB = tailB.Next
    }
    return nil
}

// 方法三: 双指针
// len(A) + len(common) + len(B) = len(B) + len(common) + len(A)
// 时间复杂度O(M+N), 空间复杂度O(1)
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    tailA, tailB := headA, headB
    // 当两者相同时, 结束循环返回
    // 即使是不相交的情况, 双方最终也会到达nil(双方互换了链)从而相等,返回的也就是nil代表不相交
    for tailA != tailB  {
        if tailA == nil {
            tailA = headB
        }else {
            tailA = tailA.Next
        }

        if tailB == nil {
            tailB = headA
        }else{
            tailB = tailB.Next      
        }      
    }
    return tailA
}
```

二刷出现的问题：双指针法中**使用如下的方式会出现问题**：

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    if headA == nil || headB == nil {
        return nil
    }

    p1, p2 := headA, headB
    for p1 != p2 {
        p1 = p1.Next
        if p1 == nil {
            p1 = headB
        }

        p2 = p2.Next
        if p2 == nil {
            p2 = headA
        }
    }

    return p1
}
```

如果两个链表不相交就会出现死循环，原因在于：一旦一个指针变为`nil`就被赋值到另一个的头节点，永远也不会出现`nil==nil`而退出循环，所以要按照上面正确的解释

三刷：

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    p1, p2 := headA, headB
    // 记得保持一下nil==nil的状态
    for p1 != p2 {
        if p1 != nil {
            p1 = p1.Next
        }else {
            p1 = headB
        }
        
        if p2 != nil {
            p2 = p2.Next
        }else {
            p2 = headA
        }
    }

    return p1
}
```
