### 1.模块缺失问题

![](http://qapzfpul3.bkt.clouddn.com/qiniu_picGo/`7FY1HL0S5V_2VOWEU9~_@M.png)

这个代码复制要注意细节啊，第一行jsonContent:   前面没有空格，后面meta到posts都是前面一个空格，在后面是两个空格。

```yml
jsonContent:
 meta: false
 pages: false
 posts:
  title: true
  date: true
  path: true
  text: false
  raw: false
  content: false
  slug: false
  updated: false
  comments: false
  link: false
  permalink: false
  excerpt: false
  categories: false
  tags: true
```

**注意你的nodejs版本是否符合要求，以及你的npm版本是否过低！**
我就是npm版本过低一直未解决，错误截图：
![](http://qapzfpul3.bkt.clouddn.com/qiniu_picGo/20200524155219.png)

使用`cnpm install -g npm@latest`更新成为最新版！！再重新
`npm i  hexo-generator-json-content --save`即可！

具体详情：https://github.com/litten/hexo-theme-yilia/issues/869

### 2.把github pages的域名设置成自己的域名
首先（我是在阿里云上注册的，以此为例）去阿里云找到域名管理，为自己域名添加解析记录：

![](http://qapzfpul3.bkt.clouddn.com/qiniu_picGo/$QZ$JZ`F_L~96LD~G[I`1UB.png)

然后打开自己仓库，setting找到github pages，设置好刚刚的域名，对应起来：

![](http://qapzfpul3.bkt.clouddn.com/qiniu_picGo/[HA42O}QDMAL3W8NY$4YLR8.png)

还有一件**重要**的事：每次`hexo g -d`会把设置清除掉，导致自己的域名无法使用

解决方法：在**根目录**下的source文件夹下新建一个名叫**CNAME**（全大写，无后缀），里面只要写自己的域名（不需要http前缀）即可。

这样就ok啦！



### 3.解决PicGo剪贴板的截图无法快速上传的问题
问题截图：

![](http://qapzfpul3.bkt.clouddn.com/qiniu_picGo/20200523201752.png)

配置环境变量（用户变量和环境变量都配置上）：
`C:\Windows\System32\WindowsPowerShell\v1.0\`

原因：picGo会调用系统的powershell.exe

具体问题解决：https://github.com/Molunerfinn/PicGo/issues/238

不会的多搜搜，你遇到的别人也会遇到，一定会找到答案的！

### 4.yilia文章太长截断（摘要）
使用` <!-- more -->`（more前后有空格），表示在当前位置截断。

在yilia/_config.yml 文件修改
```yml
#excerpt_link: more
```
对就是把这个**注释**掉，不然会重复  more>> 和 阅读全文
